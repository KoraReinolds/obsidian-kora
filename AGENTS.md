# AGENTS.md — obsidian-kora

Контекст для агентов при работе с этим репозиторием (плагин Kora для Obsidian).

## JSDoc и комментарии к коду

- Текстовые описания в JSDoc (**`@description`**, **`@param`**, **`@returns`**, **`@module`**, блоки `@typedef` и т.д.) пишутся **на русском**, если только в задаче явно не требуется иной язык.
- Имена идентификаторов из кода, типы TypeScript, цитаты из внешних API и путей к файлам оставлять **как в исходниках** (латиница).
- Краткость: по смыслу и граничным случаям; не дублировать очевидное имя функции.

Подробнее см. правило [.cursor/rules/jsdoc-language.mdc](.cursor/rules/jsdoc-language.mdc).

## Безопасное редактирование кириллицы

- Блоки с кириллицей в `.ts`, `.tsx`, `.js`, `.jsx`, `.vue`, `.md`, `.json` редактировать только способами, которые гарантированно сохраняют UTF-8 без перекодирования.
- Предпочтительный способ ручных правок: `apply_patch`.
- Если используется скриптовая правка, файл нужно читать и писать с явным `utf8`.
- Не использовать shell one-liners, массовые replace через терминал, PowerShell-конкатенацию и другие потоковые правки для строк с кириллицей, если нет отдельной уверенности в сохранении UTF-8.
- Если в файле уже появились строки вида `Р...`, `С...`, `Ð...`, `Ñ...` или `????`, это признак порчи кодировки; такие изменения нельзя оставлять как есть.
